# Frontend Design

Generates distinctive, production-grade frontend interfaces that avoid generic AI aesthetics.

## What It Does

Automatically use this skill for frontend work. Creates production-ready code with:

- **Bold aesthetic choices** - Avoid generic, template-like designs
- **Distinctive typography and color palettes** - Create unique visual identities
- **High-impact animations and visual details** - Add polish and delight
- **Context-aware implementation** - Tailor design to the specific use case

## Usage Examples

- "Create a dashboard for a music streaming app"
- "Build a landing page for an AI security startup"
- "Design a settings panel with dark mode"
- "Make a user profile card with modern aesthetics"
- "Create a navigation menu with smooth animations"

## Design Principles

1. **Avoid Generic AI Aesthetics**
   - Don't use the same gradients everyone uses
   - Avoid overused color combinations (purple-to-pink gradients)
   - Create distinctive visual identities

2. **Typography Matters**
   - Choose fonts that match the brand personality
   - Establish clear hierarchy
   - Use appropriate line heights and letter spacing

3. **Color with Purpose**
   - Select color palettes that evoke the right emotions
   - Ensure sufficient contrast for accessibility
   - Use color to guide user attention

4. **Animation & Micro-interactions**
   - Add subtle transitions that feel natural
   - Use motion to provide feedback
   - Don't over-animate - quality over quantity

5. **Component Consistency**
   - Maintain design system consistency
   - Reuse patterns appropriately
   - Document design decisions

## Implementation Guidelines

- Always consider the target platform (web, mobile, desktop)
- Ensure responsive design where applicable
- Follow accessibility best practices (WCAG guidelines)
- Optimize for performance (avoid heavy animations on low-end devices)
- Test across different browsers and screen sizes

## Output Format

When implementing designs, provide:
1. Clean, well-structured code
2. Comments explaining design choices
3. Responsive breakpoints if applicable
4. Animation/transition specifications
5. Color palette and typography details
