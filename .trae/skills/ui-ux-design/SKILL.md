# UI/UX Design

Provides comprehensive UI/UX design guidance, user research methods, and interaction design patterns.

## What It Does

Helps create user-centered designs by focusing on:

- **User Research & Analysis** - Understand user needs and behaviors
- **Information Architecture** - Organize content logically and intuitively
- **Interaction Design** - Design smooth, intuitive user flows
- **Usability & Accessibility** - Ensure products work for everyone
- **Design Systems** - Maintain consistency across products

## Usage Examples

- "Design the user flow for a checkout process"
- "Create wireframes for a mobile app onboarding"
- "Analyze the usability of this form design"
- "Suggest improvements for this navigation structure"
- "Design an accessible color scheme"
- "Create a design system for our product"

## Core Principles

### 1. User-Centered Design
- Start with user research and personas
- Design for real user needs, not assumptions
- Test with actual users early and often
- Iterate based on feedback

### 2. Clarity & Simplicity
- Remove unnecessary complexity
- Use familiar patterns when appropriate
- Provide clear feedback for all actions
- Progressive disclosure - show information when needed

### 3. Consistency
- Maintain visual consistency (colors, typography, spacing)
- Use consistent interaction patterns
- Follow platform conventions (iOS, Android, Web)
- Document design decisions in a style guide

### 4. Accessibility (A11y)
- Ensure sufficient color contrast (WCAG AA minimum)
- Support keyboard navigation
- Provide alternative text for images
- Design for screen readers
- Test with assistive technologies

### 5. Feedback & Affordance
- Show system status clearly
- Provide immediate feedback for user actions
- Use visual cues to indicate interactivity
- Error prevention and recovery

## Design Process

### 1. Discovery & Research
- User interviews and surveys
- Competitive analysis
- User personas and journeys
- Problem definition

### 2. Information Architecture
- Content inventory and audit
- Site/app mapping
- Navigation design
- Taxonomy and labeling

### 3. Wireframing & Prototyping
- Low-fidelity sketches
- Mid-fidelity wireframes
- High-fidelity prototypes
- Interactive prototypes for testing

### 4. Visual Design
- Color palette selection
- Typography system
- Iconography and imagery
- Component library

### 5. Testing & Iteration
- Usability testing
- A/B testing
- Analytics review
- Continuous improvement

## Common Patterns

### Navigation
- Bottom navigation (mobile)
- Sidebar navigation (desktop/web apps)
- Tab bars for categorized content
- Breadcrumbs for deep hierarchies
- Search and filters

### Forms & Input
- Inline validation
- Clear error messages
- Smart defaults
- Multi-step forms for complex data
- Auto-save functionality

### Feedback
- Toast notifications
- Modal dialogs for critical actions
- Inline confirmation messages
- Loading states and skeleton screens
- Empty states

### Data Display
- Tables with sorting and filtering
- Cards for content browsing
- Lists for sequential content
- Charts and visualizations
- Dashboards with key metrics

## Platform Guidelines

### iOS (Human Interface Guidelines)
- Respect safe areas and notches
- Use SF Symbols for icons
- Follow iOS navigation patterns
- Support Dark Mode
- Consider Dynamic Type

### Android (Material Design)
- Use Material Design components
- Support different screen densities
- Follow Android navigation patterns
- Implement proper touch targets (48dp minimum)
- Support system themes

### Web
- Responsive design principles
- Cross-browser compatibility
- Performance optimization
- SEO considerations
- Progressive enhancement

## Tools & Resources

- **Prototyping**: Figma, Principle, ProtoPie, Framer
- **User Testing**: Maze, UserTesting, Hotjar
- **Design Systems**: Storybook, Zeroheight

1. User flow diagrams or descriptions
2. Wireframe descriptions or sketches
3. Design rationale and decisions
4. Accessibility considerations
5. Testing recommendations
