## 快速参考

### 项目架构

```
┌─────────────────────────────────────────┐
│              皮肤层 (Skin)               │  theme/
│         主题系统、样式变量、动态换肤        │
├─────────────────────────────────────────┤
│             结构层 (Structure)           │  pages/
│         页面结构、组件规范、布局系统        │
├─────────────────────────────────────────┤
│              逻辑层 (Logic)              │  services/
│         业务逻辑、API服务、数据处理        │
├─────────────────────────────────────────┤
│            数据互通层 (Data)             │  stores/ config/
│         全局配置、状态管理、常量定义        │
├─────────────────────────────────────────┤
│           微信云开发 (Cloud Base)         │  cloudfunctions/
│         云函数、云数据库、云存储           │
└─────────────────────────────────────────┘
```

### 技术栈

| 层级 | 技术 | 说明 |
|-----|------|------|
| 前端框架 | 微信小程序原生 | WXML + WXSS + JavaScript |
| 后端服务 | 微信云开发 | 云函数 + 云数据库 + 云存储 |
| 数据库 | MongoDB | 文档型 NoSQL 数据库 |
| 运行环境 | Node.js | 云函数运行环境 |

### 命名速查

| 类型 | 规范 | 示例 |
|------|------|------|
| 文件 | 小写 + 连字符 | `user-service.js` |
| 变量 | 小驼峰 | `userName` |
| 常量 | 全大写 + 下划线 | `MAX_RETRY_COUNT` |
| 类 | 大驼峰 | `UserService` |
| 函数 | 动词 + 名词 | `getUserInfo()` |
| API | `资源_动作_操作` | `user_login` |

### Git 提交速查

```bash
# 新功能
git commit -m "feat(用户): 添加微信一键登录功能"

# Bug 修复
git commit -m "fix(预约): 修复时间冲突检测逻辑"

# 文档更新
git commit -m "docs: 更新 API 接口文档"

# 代码格式
git commit -m "style: 统一代码缩进格式"

# 重构
git commit -m "refactor(用户): 重构用户认证逻辑"

# 性能优化
git commit -m "perf(列表): 优化会议室列表加载速度"
```

### 分支管理速查

```bash
# 创建功能分支
git checkout dev
git pull origin dev
git checkout -b feat/user-login

# 提交代码
git add .
git commit -m "feat(用户): 实现微信登录功能"
git push origin feat/user-login

# 合并后清理
git checkout dev
git pull origin dev
git branch -d feat/user-login
git push origin --delete feat/user-login
```

## 代码审查检查清单

### 基础规范

- [ ] 代码符合命名规范（变量、函数、文件）
- [ ] 代码缩进统一（2 个空格）
- [ ] 必要的注释已添加（文件头、函数、复杂逻辑）
- [ ] 无 `console.log` 等调试代码残留
- [ ] 无硬编码的敏感信息（密钥、密码等）

### 架构规范

- [ ] 遵循四层架构，层间依赖正确
- [ ] 业务逻辑封装在服务层，不在页面中直接处理
- [ ] 使用 UserStore 管理全局用户信息
- [ ] 页面正确引入 theme-mixin 支持主题切换
- [ ] 云函数使用处理器映射模式
- [ ] 权限检查使用统一的 permission.js 工具

### 功能性

- [ ] 功能实现符合需求文档
- [ ] 边界条件已处理（空值、越界等）
- [ ] 错误处理已完善（try-catch、错误提示）
- [ ] 异步操作已正确处理（async/await）

### 安全性

- [ ] 用户输入已验证
- [ ] 敏感数据已加密存储
- [ ] 权限控制已正确实现
- [ ] 云函数权限已配置

### 性能

- [ ] 无明显的性能问题（循环嵌套、重复计算）
- [ ] 列表数据已考虑分页
- [ ] 图片等资源已优化（压缩、CDN）
- [ ] 避免不必要的 setData
