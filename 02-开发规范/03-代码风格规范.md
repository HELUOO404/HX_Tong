## 一、基本格式规范

### 1.1 缩进与换行

- **缩进**：使用 2 个空格（禁止使用 Tab）
- **换行**：Unix 风格换行符（LF）
- **行尾**：语句末尾不使用分号（JavaScript）
- **行长**：单行代码不超过 100 个字符
- **空行**：
  - 函数之间保留 1 个空行
  - 逻辑块之间保留 1 个空行
  - 文件末尾保留 1 个空行

```javascript
// 正确示例
Page({
  data: {
    userInfo: null,
    meetingRooms: []
  },

  onLoad() {
    this.initData()
  },

  initData() {
    this.fetchMeetingRooms()
  }
})
```

### 1.2 引号使用

- **字符串**：统一使用单引号 `'string'`
- **模板字符串**：使用反引号 `` `template` ``
- **JSON**：使用双引号（符合 JSON 标准）

```javascript
// 正确示例
const name = '红芯通'
const greeting = `欢迎使用${name}`
const config = { "key": "value" }
```

### 1.3 括号与空格

```javascript
// 正确示例
if (condition) {
  doSomething()
}

function foo(a, b) {
  return a + b
}

// 对象定义
const obj = {
  key: 'value',
  num: 123
}

// 数组定义
const arr = [1, 2, 3]

// 函数调用
foo(arg1, arg2)
```

### 1.4 注释规范

#### 文件头注释

每个文件必须包含文件头注释：

```javascript
/**
 * @file 用户服务模块
 * @description 处理用户相关的业务逻辑，包括登录、信息获取、权限验证等
 * @author 红芯通开发团队
 * @since 2026-04-22
 * @version 1.0.0
 */
```

#### 函数/方法注释

所有公共函数必须包含 JSDoc 注释：

```javascript
/**
 * 预约会议室
 * @param {Object} bookingInfo - 预约信息
 * @param {string} bookingInfo.roomId - 会议室 ID
 * @param {string} bookingInfo.date - 预约日期 (YYYY-MM-DD)
 * @param {string} bookingInfo.startTime - 开始时间 (HH:MM)
 * @param {string} bookingInfo.endTime - 结束时间 (HH:MM)
 * @returns {Promise<Object>} 返回预约结果
 * @throws {Error} 当时间段冲突时抛出异常
 * @example
 * const result = await bookMeetingRoom({
 *   roomId: 'room_001',
 *   date: '2026-04-22',
 *   startTime: '09:00',
 *   endTime: '11:00'
 * })
 */
async function bookMeetingRoom(bookingInfo) {
  // 实现逻辑
}
```

#### 行内注释

```javascript
// 正确示例：解释"为什么"而不是"做了什么"
// 由于微信云开发限制，单次查询最多返回 100 条记录
const MAX_QUERY_LIMIT = 100

// 错误示例：重复代码本身的内容
const count = 10 // 设置计数为 10
```

#### TODO/FIXME 标记

```javascript
// TODO: 后续版本需要接入学校统一身份认证
// FIXME: 当前实现存在性能问题，需要优化
```

## 二、JavaScript 规范

### 2.1 变量声明

- 使用 `const` 声明常量
- 使用 `let` 声明变量
- **禁止**使用 `var`

```javascript
// 正确示例
const API_BASE = 'https://api.example.com'
let count = 0

// 错误示例
var name = 'test'
```

### 2.2 函数定义

- 优先使用箭头函数
- 类方法使用简写语法
- 回调函数使用箭头函数

```javascript
// 正确示例
const add = (a, b) => a + b

const obj = {
  value: 10,
  
  // 方法简写
  getValue() {
    return this.value
  },
  
  // 箭头函数用于回调
  async fetchData() {
    const result = await api.getData().then(res => res.data)
    return result
  }
}

// 错误示例
const add = function(a, b) {
  return a + b
}
```

### 2.3 异步处理

- 使用 `async/await` 处理异步操作
- 使用 `try/catch` 捕获错误
- 避免回调地狱

```javascript
// 正确示例
async function loadData() {
  try {
    const userInfo = await userService.getInfo()
    const bookings = await bookingService.getList(userInfo.id)
    this.setData({ bookings })
  } catch (error) {
    console.error('加载数据失败:', error)
    wx.showToast({ title: '加载失败', icon: 'none' })
  }
}

// 错误示例
function loadData() {
  userService.getInfo().then(userInfo => {
    bookingService.getList(userInfo.id).then(bookings => {
      this.setData({ bookings })
    }).catch(error => {
      console.error(error)
    })
  })
}
```

### 2.4 对象与数组

- 使用对象简写语法
- 使用展开运算符复制对象/数组
- 使用解构赋值

```javascript
// 正确示例
const name = '红芯通'
const version = '1.0.0'

// 对象简写
const app = { name, version }

// 展开运算符
const newApp = { ...app, updatedAt: Date.now() }
const newArr = [...oldArr, newItem]

// 解构赋值
const { userInfo, isLogin } = this.data
const [first, second] = list

// 错误示例
const app = { name: name, version: version }
```

### 2.5 条件判断

- 使用严格相等 `===` 和 `!==`
- 简化布尔判断
- 提前返回减少嵌套

```javascript
// 正确示例
if (isLogin === true) {
  // ...
}

// 简化布尔判断
if (isLogin) {
  // ...
}

// 提前返回
function processData(data) {
  if (!data) return null
  if (!data.items) return []
  
  return data.items.map(item => processItem(item))
}

// 错误示例
if (isLogin == true) {
  // ...
}
```

### 2.6 错误处理

- 统一错误处理机制
- 错误信息清晰明确
- 记录错误日志

```javascript
// 正确示例
async function submitForm() {
  try {
    await api.submit(this.data.form)
    wx.showToast({ title: '提交成功', icon: 'success' })
  } catch (error) {
    console.error('[SubmitForm] 提交失败:', error)
    wx.showToast({ 
      title: error.message || '提交失败，请重试', 
      icon: 'none' 
    })
  }
}
```

## 三、WXML 规范

### 3.1 标签规范

- 标签名小写
- 属性使用双引号
- 自闭合标签

```xml
<!-- 正确示例 -->
<view class="container">
  <image src="{{userInfo.avatar}}" mode="aspectFill" />
  <text>{{userInfo.name}}</text>
</view>

<!-- 错误示例 -->
<view class='container'>
  <image src='{{userInfo.avatar}}'></image>
</view>
```

### 3.2 表达式限制

WXML 中的双大括号 `{{}}` 只能使用简单的表达式，**不支持**以下 JavaScript 特性：

| 不支持的功能 | 错误示例 | 正确做法 |
|-------------|---------|---------|
| 数组方法 | `{{arr.findIndex(item => item.id === id)}}` | 在 JS 中计算，通过 data 绑定结果 |
| 箭头函数 | `{{arr.filter(item => item.active)}}` | 在 JS 中预处理数据 |
| 可选链操作符 | `{{obj?.property?.value}}` | 使用 `{{obj.property.value}}` 或在 JS 中处理 |
| 空值合并操作符 | `{{value ?? defaultValue}}` | 使用三元运算符 `{{value ? value : defaultValue}}` |
| 复杂运算 | `{{a + b * c / d}}` | 在 JS 中计算结果 |
| 三元嵌套 | `{{a ? b ? c : d : e}}` | 简化逻辑或在 JS 中处理 |
| 正则表达式 | `{{str.replace(/\d/g, '')}}` | 在 JS 中使用 wxs 或预处理 |
| JSON 方法 | `{{JSON.parse(str)}}` | 在 JS 中解析后存入 data |

```xml
<!-- 错误：使用 findIndex -->
<picker range="{{options}}" value="{{options.findIndex(item => item.value === selectedValue)}}">

<!-- 正确：使用索引值 -->
<picker range="{{options}}" range-key="label" value="{{selectedIndex}}">
```

### 3.3 条件渲染

```xml
<!-- 正确示例 -->
<view wx:if="{{isLogin}}">
  <text>已登录</text>
</view>
<view wx:else>
  <text>未登录</text>
</view>

<!-- 列表渲染 -->
<view 
  wx:for="{{roomList}}" 
  wx:key="_id"
  wx:for-item="room"
  wx:for-index="index"
>
  <text>{{room.name}}</text>
</view>
```

## 四、WXSS 规范

### 4.1 选择器规范

- 使用类选择器
- 避免使用 ID 选择器
- 避免嵌套过深

```css
/* 正确示例 */
.room-card {
  padding: 16px;
  background: #fff;
}

.room-card .title {
  font-size: 16px;
}

/* 错误示例 */
#roomCard {
  padding: 16px;
}

.room-card .content .header .title .text {
  font-size: 16px;
}
```

### 4.2 属性顺序

```css
/* 推荐顺序：
  1. 布局属性（position, display, flex 等）
  2. 盒模型属性（width, height, padding, margin 等）
  3. 视觉属性（background, border, color 等）
  4. 文本属性（font, text-align 等）
  5. 其他属性
*/
.card {
  /* 布局 */
  display: flex;
  flex-direction: column;
  
  /* 盒模型 */
  width: 100%;
  padding: 16px;
  margin-bottom: 12px;
  
  /* 视觉 */
  background: #fff;
  border-radius: 8px;
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.1);
  
  /* 文本 */
  font-size: 14px;
  color: #333;
}
```

### 4.3 单位使用

- 使用 `rpx` 进行屏幕适配
- 字体大小使用 `px` 或 `rpx`
- 边框使用 `px`

```css
/* 正确示例 */
.container {
  padding: 32rpx;
}

.title {
  font-size: 36rpx;
  line-height: 1.5;
}

.border {
  border: 1px solid #e0e0e0;
}

/* 错误示例 */
.container {
  padding: 16px;
}
```

### 4.4 颜色规范

- 使用主题变量
- 避免硬编码颜色值

```css
/* 正确示例（使用主题变量） */
.card {
  background: {{theme.card}};
  color: {{theme.textPrimary}};
}

/* 错误示例（硬编码） */
.card {
  background: #fff;
  color: #333;
}
```
