## 四、文件存放规则

### 4.1 禁止行为

| 禁止行为 | 说明 | 正确做法 |
|---------|------|---------|
| 在根目录放置脚本文件 | 根目录只允许保留配置文件 | 所有脚本放入 `04-工具脚本/` |
| 在云函数中直接操作数据库 | 必须使用封装的数据访问层 | 使用统一的工具函数 |
| 在页面中直接调用 `wx.cloud` | 必须通过服务层调用 | 使用 `services/` 下的服务类 |
| 散落在代码中的魔法数字 | 所有常量必须定义在配置文件中 | 使用 `config/constants.js` |
| 在源代码目录存放中间文件 | 中间文件必须放入指定目录 | 使用 `.trae/temp/` 或 `04-工具脚本/` |

### 4.2 文件类型存放位置

| 文件类型 | 存放位置 | 示例 |
|---------|---------|------|
| 页面文件 | `pages/{page}/` | `pages/index/index.js` |
| 组件文件 | `components/{type}/{name}/` | `components/common/button/` |
| 服务类 | `services/` | `services/userService.js` |
| 工具函数 | `utils/` | `utils/errorHandler.js` |
| 配置文件 | `config/` | `config/env.js` |
| 主题文件 | `theme/` | `theme/theme-config.js` |
| 状态管理 | `stores/` | `stores/userStore.js` |
| 静态资源 | `assets/images/` | `assets/images/icons/` |
| 云函数 | `cloudfunctions/{name}/` | `cloudfunctions/userService/` |
| 工具脚本 | `04-工具脚本/` | `04-工具-scripts/assets/` |

## 六、中间文件管理

### 6.1 中间文件定义

中间文件是指脚本执行、构建过程、工具操作产生的临时或输出文件：
- 脚本执行产生的输出文件
- 临时生成的缓存文件
- 构建过程中间产物
- 工具生成的资源文件
- 测试数据文件
- 日志文件

### 6.2 中间文件存储位置

```
HX_Tong/
├── .trae/
│   └── temp/                               # Trae 相关临时文件
│       ├── output/                         # 脚本输出
│       ├── cache/                          # 缓存文件
│       └── logs/                           # 日志文件
├── 04-工具脚本/
│   ├── assets/
│   │   └── output/                         # 资源生成脚本输出
│   ├── database/
│   │   └── output/                         # 数据库脚本输出
│   └── utils/
│       └── fileHelper.js                   # 文件操作工具
└── .gitignore                              # 忽略中间文件
```

### 6.3 中间文件规则

| 文件类型 | 存储位置 | 清理策略 |
|---------|---------|---------|
| 图标生成输出 | `04-工具脚本/assets/output/` | 每次执行前清空 |
| 图片转换输出 | `04-工具脚本/assets/output/` | 保留最近 10 次 |
| 数据库初始化输出 | `04-工具脚本/database/output/` | 保留最近 5 次 |
| 主题生成输出 | `.trae/temp/output/themes/` | 版本控制 |
| 构建缓存 | `miniprogram/.temp/` | 构建完成后清理 |
| 下载缓存 | `.trae/temp/cache/` | 7 天自动清理 |
| 日志文件 | `.trae/temp/logs/` | 30 天自动清理 |

## 八、文件组织检查清单

- [ ] 服务类存放在 `services/` 目录
- [ ] 工具函数存放在 `utils/` 目录
- [ ] 配置文件存放在 `config/` 目录
- [ ] 主题文件存放在 `theme/` 目录
- [ ] 状态管理存放在 `stores/` 目录
- [ ] 云函数存放在 `cloudfunctions/` 目录
